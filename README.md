# Promptex
A beginner-friendly cheat sheet for prompt engineering techniques.

## 📚 Prompt Engineering Techniques — Master Summary

| # | Technique | Description |
|---|-----------|-------------|
| 1 | 🎯 **Zero-Shot** | Asking AI to complete a task without giving any examples first. The AI uses its existing training to figure it out. Your go-to everyday technique.<br><br>*Example: "Write a short funny out-of-office email reply for my colleagues."* |
| 2 | 🎨 **CRAFT Framework** | A 5-ingredient formula for building perfect prompts. Combine all five for maximum results. |
| 2.1 | ↳ **C — Context** | Your background situation — gives the AI the info it needs about YOU.<br><br>*Example: "I'm a complete beginner, I'm tired, I only have 30 minutes in the morning."* |
| 2.2 | ↳ **R — Role** | Who the AI should be — sets expertise and perspective.<br><br>*Example: "You are a professional fitness trainer."* |
| 2.3 | ↳ **A — Action** | What you want done — always starts with an imperative verb.<br><br>*Example: "Create a 7-day workout plan."* |
| 2.4 | ↳ **F — Format** | How the response should look — structure, length, layout.<br><br>*Example: "Format as a markdown table, max 100 words per day."* |
| 2.5 | ↳ **T — Tone** | How the response should sound — the voice and style.<br><br>*Example: "Keep it simple, concise, and friendly for beginners."* |
| 3 | 🚂 **Prefixing** | You start the sentence, the AI completes it. Instantly steers the direction and format of the response.<br><br>*Example: "The three biggest reasons people fail at dieting are: 1."* |
| 4 | ❓ **Questions** | Using H-questions — Who, What, When, Where, Why, How — as natural zero-shot prompts. Simple, clean, and effective.<br><br>*Example: "How does blockchain work?"* |
| 5 | 💪 **Commands** | Starting prompts with strong action verbs. Direct, clear, and powerful — no fluff needed.<br><br>*Example: "List 5 ways to save money. / Explain how vaccines work. / Compare iPhone vs Android."* |
| 6 | 🧠 **Chain of Thought (CoT)** | Instructing the AI to reason step by step before answering. Produces deeper, more accurate, well-structured responses.<br><br>*Example: "What's the best business to start with $1000? Think through this step by step, considering budget, time, and market demand."* |
| 7 | 🎤 **Clarifying Questions** | You instruct the AI to interview YOU before starting the task. The AI gathers missing information first, then produces a perfectly tailored response.<br><br>*Example: "You are a professional party planner. Before creating my party plan, ask me all the questions you need to create the perfect result."* |
| 8 | 🖼️ **Few-Shot** | Giving the AI 2–3 examples of what you want BEFORE asking it to do the task. Like showing someone how to draw a cat before asking them to draw one. The AI learns your pattern and copies it.<br><br>**Example 1 — Formal to Casual Translator:**<br>- "I require assistance." → "I need help."<br>- "Please be advised that the meeting has been postponed." → "Heads up — the meeting got pushed back."<br><br>Now translate: *"I would like to inquire about the status of my application."*<br><br>**Example 2 — Email Tone Detector:**<br>- "Per my last email..." → 😤 Passive-aggressive<br>- "Hope this helps!" → 😊 Friendly<br><br>Now classify: *"As previously stated, the deadline was last Friday."*<br><br>**Example 3 — Bug Report Formatter:**<br>- "App crashes when I click save" → 🐛 **Bug:** App crashes \| **Trigger:** Clicking save<br>- "Dark mode breaks after update" → 🐛 **Bug:** Dark mode broken \| **Trigger:** After update<br><br>Now format: *"The search bar returns no results when I type too fast."*<br><br>💡 **Pro tip:** Combine Few-Shot + CRAFT for maximum power! |
| 9 | 🔗 **Prompt Chaining** | Breaking a big task into smaller connected prompts — where each prompt **builds on the output of the previous one.** Like a relay race 🏃‍♂️ where each runner receives the baton and carries it forward.<br><br>**The magic formula:**<br>- Prompt 1: *"Do X"*<br>- Prompt 2: *"Based on that, do Y"*<br>- Prompt 3: *"Using that result, do Z"*<br><br>**Connector words:** *"Based on that / Using that result / Now take that and / Building on that"*<br><br>*Example:*<br>- *"List the steps to build a C# REST API for a blog."*<br>- *"Using that plan, write the C# code."*<br>- *"Review the code you just wrote and fix all errors."*<br>- *"Write e2e tests for the final code."* |
| 10 | 🎬 **Session Priming** | Setting the AI's rules, role, and behavior **once at the beginning** of a session — so every response that follows is perfectly aligned. Like briefing a personal assistant before their first day! 🧑‍💼<br><br>**The CRAFT + Chaining combo:**<br>- Prompt 1 — Role: *"You are a master C# developer working for me. Agreed?"*<br>- Prompt 2 — Context: *"I am an intermediate developer building an e-commerce API. Got it?"*<br>- Prompt 3 — Format: *"Always write clean, well-commented code. Clear?"*<br>- Prompt 4 — Tone: *"Keep explanations concise, no jargon, ask clarifying questions before coding. Understood?"*<br>- Prompt 5 — Action: *"Now build me a login endpoint."* 🚀<br><br>💡 **Key rules:**<br>- Role = who the AI is 🤖<br>- Context = who YOU are 🧑‍💻<br>- Always confirm with *"Agreed / Got it / Clear?"*<br>- Action always comes LAST! |
| 11 | 🪞 **Self-Consistency** | Asking the same prompt in multiple fresh sessions and picking the most common answer. Like asking 5 different people for directions and going with the majority! 🗺️<br><br>Think of it as the "refresh button" for prompts — not happy with an answer? Open a fresh session and try again! 🔄<br><br>💡 Key rules:<br>- Always use fresh sessions — not multiple answers in one prompt 🆕<br>- Ask the exact same prompt each time 📋<br>- Pick the most common answer ✅<br>- Best for high-stakes decisions — architecture, patterns, libraries 🏗️<br><br>Example:<br>"What is the best database for a C# e-commerce app?"<br>→ Open 3 fresh sessions, ask the same question, compare results, pick the most common answer! 🏆<br><br>⚠️ Modern LLM reality: Asking for multiple answers in ONE prompt won't work — AI deliberately varies them. Always use fresh sessions for true independence! |
| 12 | ✂️ **Negative Prompting** | Telling the AI what NOT to do — just as important as telling it what TO do! Like ordering a pizza and saying "no olives, no anchovies, no thick crust!" 🍕<br><br>Your negative keywords: Don't / Avoid / Never / Without / Exclude<br><br>💡 Key rules:<br>- Always pair with a positive anchor — tell AI where to go AND where not to go 🧭<br>- Be specific — "max 500 words" beats "not too long" 🎯<br>- Watch for positives in disguise — "don't forget to..." is actually a command! ⚠️<br>- Combine with Session Priming for maximum control 🎬<br><br>Example:<br>"Help me design a simple, scalable C# microservices architecture.<br>- Don't suggest overcomplicated solutions.<br>- Don't use .NET features my team hasn't adopted.<br>- Don't write explanations before code — always after.<br>- Don't recommend libraries without explaining why.<br>- Don't assume I work alone — we're a team of 5." 🎯 |
| 13 | 🌳 **Tree of Thought** | Instead of thinking in a straight line, the AI **explores multiple solution branches simultaneously**, evaluates each one, and picks the best! Like a chess player ♟️ thinking through all possible moves before deciding.<br><br>**The magic 3-step formula:**<br>- 🌿 Explore: *"Explore X options..."*<br>- ⚖️ Evaluate: *"For each, list pros and cons..."*<br>- 🏆 Recommend: *"Then recommend the best one..."*<br><br>*Example:*<br>*"Explore the 5 most popular e-commerce platforms for .NET, evaluate each step by step by price, quality, documentation and support, list pros and cons for each, then recommend the best one with an explanation why."* 🌳<br><br>💡 **Key differences:**<br>- vs 🧠 CoT: CoT follows ONE path, ToT explores MANY 🌳<br>- vs 🪞 Self-Consistency: ToT works in ONE session, Self-Consistency needs multiple 🆕<br><br>⚡ **Power combo:** Session Priming + Negative Prompting + Tree of Thought + CoT guidance = the ultimate high-stakes decision prompt! 🏆 |
| 14 | 🧬 **Meta Prompting** | Instead of giving specific examples like Few-Shot, you give the AI an **abstract template** — teaching it the shape of the solution rather than the solution itself! Like teaching someone **how to paint** instead of showing them 3 paintings. 🎨<br><br>**The magic formula:**<br>- *"Do [task] using this structure:"*<br>- *[Element 1]: [placeholder]*<br>- *[Element 2]: [placeholder]*<br>- *[Element 3]: [placeholder]*<br>- *"Now apply this to: [your input]"*<br><br>*Example:*<br>*"Review any C# code using this structure:*<br>- *Code Quality: [what's good, what's bad]*<br>- *Performance: [bottlenecks and suggestions]*<br>- *Security: [vulnerabilities found]*<br>- *Improvements: [specific actionable fixes]*<br><br>*Now review this code: ..."* 🧬<br><br>💡 **Key advantages over Few-Shot:**<br>- Uses fewer tokens — no need for full examples 📉<br>- Works for ANY input — fully reusable ♻️<br>- More flexible — not tied to specific cases 🔄<br><br>⚠️ **When NOT to use it:** Novel or unique tasks where the AI needs concrete examples to understand what you want — use Few-Shot instead! |
